API integration

The backend exposes a small set of JSON endpoints. All authenticated endpoints rely on the vlx_session cookie set at login — no separate API token is required for first-party use.

POST /api/register

{ "firstName": "Jane", "lastName": "Doe", "email": "jane@example.com", "password": "••••••••" }

Returns 201 and sets a session cookie on success, 409 if the email is already registered.

POST /api/login

{ "email": "jane@example.com", "password": "••••••••" }

Returns 200 and a session cookie on success, 401 on incorrect credentials.

GET /api/me

Returns the authenticated user's profile, or 401 if there is no valid session.

GET /api/prices

Returns live market prices, proxied server-side from CoinGecko's public API (cached 60 seconds).

Personal API keys (for your own bots)

The dashboard's API key page is ready in the UI. Wire it to real, scoped tokens by extending server.js with an api_keys store and checking an Authorization: Bearer header alongside the cookie check already in place.
