Deployment

Since server.js is plain Node with no dependencies, it runs on any host that supports Node 18+.

VPS (with a process manager)

npm install -g pm2
pm2 start server.js --name vaultex
pm2 save

Put nginx or Caddy in front for HTTPS and to serve as a reverse proxy.

Render / Railway / Fly.io

Point the platform's start command at node server.js — no build command is needed.

Docker

FROM node:18-slim
WORKDIR /app
COPY . .
CMD ["node", "server.js"]

Before going live

  • Set the SESSION_SECRET environment variable explicitly, rather than relying on the auto-generated file — especially if you run more than one server instance.
  • Put the app behind HTTPS. Session cookies are HttpOnly but still travel in plaintext over unencrypted HTTP.
  • Add rate limiting in front of /api/login and /api/register to slow brute-force attempts.
